# my-weather — STM32 Nucleo-F401RE dauwpuntsensor (FreeRTOS)\n\nDit is een STM32CubeIDE/FreeRTOS voorbeeldproject voor Nucleo-F401RE dat:\n- DHT22 uitleest (bitbanged met DWT microsecond delays)\n- dauwpunt berekent (Magnus-formule)\n- waarden toont op een SSD1306 OLED via u8g2 (I2C)\n\nBranch: feature/stm32-dewpoint\n\nWiring\n- DHT22:\n  - VCC -> 3.3V\n  - GND -> GND\n  - DATA -> PA0 (GPIO, met 4.7k-10k pull-up naar 3.3V)\n- SSD1306 (I2C):\n  - VCC -> 3.3V\n  - GND -> GND\n  - SDA -> PB7\n  - SCL -> PB6\n  - I2C adres: 0x3C\n\nCubeMX / STM32CubeIDE setup (kort)\n1. Nieuw project: Nucleo-F401RE.\n2. System Clock: HSI with PLL to 84 MHz (instellingen in het .ioc bestand hieronder).\n3. Peripherals:\n   - I2C1: Enable, set PB6=SCL, PB7=SDA.\n   - Configure PA0 als GPIO Output (CubeMX kan later als Input/Output worden aangepast in code).\n4. Middleware:\n   - FreeRTOS: enable (CMSIS‑RTOS2). Kies default config.\n5. Generate code naar STM32CubeIDE.\n6. Kopieer de files uit deze branch in de passende src/inc mappen van het project en voeg u8g2 library toe (zie hieronder).\n\nu8g2 integratie\n- Voeg de u8g2 library toe als submodule of plaats de benodigde u8g2 bestanden in Drivers/u8g2/.\n- De wrapper ssd1306_u8g2.c gebruikt HAL_I2C via callback; zorg dat hi2c1 beschikbaar is (gegenereerd door CubeMX).\n\nBestanden in deze branch\n- Inc/dht22.h\n- Src/dht22.c\n- Inc/ssd1306_u8g2.h\n- Src/ssd1306_u8g2.c\n- Src/main_app.c\n- my-weather.ioc\n\nBuild & Run\n1. Importeer project in STM32CubeIDE.\n2. Voeg u8g2-source files toe (of voeg u8g2 als submodule/library).\n3. Build en flash naar Nucleo-F401RE.\n4. Open serial voor debug (optioneel) of bekijk OLED.\n\nAls je akkoord geeft, ik heb deze bestanden gecommit naar feature/stm32-dewpoint.
